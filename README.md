## 🧩 Study Topics

### 🖥️ Frontend
- **HTML / CSS**: 웹 기본 구조 및 스타일링
- **JavaScript / jQuery**: DOM 제어, 이벤트 처리, 비동기 통신
- **React**: 컴포넌트 기반 SPA 구조 학습

### ⚙️ Backend
- **Java**: 객체지향 프로그래밍 기초
- **JSP & JSTL**: 동적 페이지 구성, 서버-클라이언트 데이터 전달
- **Spring Framework**: MVC 구조, 의존성 주입, JPA / MyBatis
- **Python / Jupyter**: 데이터 분석, 간단한 자동화 스크립트

### 🗄️ Database
- **SQL (DB2, MySQL, Oracle, PostgreSQL)**: 데이터 조작 및 쿼리 실습
- **NoSQL (MongoDB, Redis)**: 비정형 데이터 처리, 캐싱 활용

### ☁️ Infra & DevOps
- **Git**: 버전 관리, 커밋 규칙, 브랜치 전략
- **Docker**: 컨테이너 환경 구성
- **CI/CD**: 자동 빌드 및 배포 파이프라인
- **AWS**: 클라우드 환경 및 배포 실습

### 🐧 Linux
- 주요 명령어 정리 및 서버 관리 실습

## 🧭 Commit Convention

모든 커밋 메시지는 아래 형식을 따릅니다:

<타입>: <변경 요약>

| 타입 | 의미 |
|------|------|
| `add` | 새로운 파일/내용 추가 |
| `update` | 기존 내용 수정 |
| `fix` | 오타나 오류 수정 |
| `refactor` | 코드 구조 개선 (동작 변화 없음) |
| `remove` | 파일/내용 삭제 |
| `docs` | 문서 관련 수정 |

📄 상세 규칙은 [`04_Infra_DevOps/Git/README.md`](04_Infra_DevOps/Git/README.md) 에서 확인할 수 있습니다.
